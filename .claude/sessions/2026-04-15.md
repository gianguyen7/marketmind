## Session 21:00 — API data investigation, data availability docs, Streamlit dashboard build

### Context
User resumed from the April 12 research completion (Polymarket efficiency conclusion). Session focused on investigating Polymarket API data availability, documenting findings, and building the C2 Streamlit dashboard to present research results.

### New information
- **Gamma API** still serves closed market metadata back to January 2021, but active markets only go back to May 2025.
- **CLOB API** now enforces a **31-day rolling window** on price history. Closed/resolved markets return empty `{"history": []}`. The `interval=max` parameter still works syntactically but is silently capped at ~31 days.
- **Our dataset is irreplaceable.** The 640K snapshots (Nov 2022 – Apr 2026) spanning full market lifetimes cannot be re-fetched. 1,879 markets have snapshot spans > 31 days; 43 span > 1 year. Max span is 674 days.
- **Root cause: CLOB v2 migration.** Polymarket announced a full exchange overhaul on April 6, 2026 (CTF Exchange V2, new collateral token Polymarket USD, rebuilt CLOB). Our fetch on ~April 9 captured data from the old CLOB v1 backend just before the cutover.
- No official announcement, press release, or changelog entry documents the price history truncation. It is an undocumented side effect of the migration.
- GitHub issue #216 (Dec 2025) already showed `interval=max` returning empty for resolved markets at fine granularity — data retention was tightening before the v2 migration.
- Polymarket's April 9 changelog changed `closed` query param default to `false` on `GET /markets`.
- April 10 changelog introduced `GET /markets/keyset` and `GET /events/keyset` for cursor-based pagination, deprecating offset-based endpoints.
